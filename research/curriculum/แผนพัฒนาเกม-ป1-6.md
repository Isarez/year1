# แผนพัฒนาเกม ป.1-6 — Roadmap แบ่งทีละระดับชั้น + phase

**ไฟล์นี้คือ:** แผนการพัฒนาเกมจากผลวิจัยหลักสูตร ป.1-6 (`หลักสูตรที่ดีที่สุด.md` ทั้ง 6 ชั้น + `แผนเกมภาพรวม-ป1-6.md`) — จัดลำดับเป็น roadmap ที่ลงมือทำได้จริง โดยยึด **"ทำทีละระดับชั้น"** เป็นแกนหลัก และแตกแต่ละชั้นเป็น **phase** (ก้อนที่ release ได้จริง 1 รอบ) เมื่อ feature ใหญ่เกินทำรวดเดียว

**สถานะ:** เป็น**แผน/ข้อเสนอ** ยังไม่ใช่สิ่งที่ตัดสินใจทำแล้ว — แต่ละ phase ต้องผ่าน IDEA.md → FEATURE.md → cf ตามกติกาโปรเจกต์ก่อนเริ่มพัฒนาจริงเสมอ และทุกเกม/หมวดใหม่ต้องสร้าง branch `feature/<ชื่อ>` แยก + ติดป้าย `isNew:true` ตามกฎใน CLAUDE.md

---

## หลักการจัดแพลน (อ่านก่อน)

1. **แกนหลัก = ทีละระดับชั้น** ตามที่ผู้ใช้กำหนด — ทำ ป.1 ให้จบก่อน แล้วค่อย ป.2 → ป.6 เรียงลำดับ (ป.1 อยู่ก่อนเพราะใกล้เนื้อหาแอปปัจจุบันที่สุด อายุ 6-7 ต่อจากกลุ่มเป้าหมายเดิม 5-6 ปีพอดี) — ผู้ใช้จะสั่งทำทีละ feature
2. **แต่ละ phase = 1 ก้อนที่เล่นจบ/ทดสอบได้ (merge สะสม) แต่ไม่ deploy รายเฟส** — ตัดสินใจแล้ว (2026-07-20): พัฒนาทีละ phase/feature แล้ว **สะสมทุกชั้นทุกเฟส deploy ทีเดียวตอนจบครบทั้ง 6 ชั้น** เป็นการอัปเกรด webapp ครั้งใหญ่ครั้งเดียว (version bump + changelog + `gh release` + flip badge ทำครั้งเดียวตอนจบ ไม่ทำระหว่างทาง)
3. **เกณฑ์แตก phase**: (ก) กลุ่ม "ต่อยอดของเดิม" ที่ใช้ mechanic เดิม = quick win รวมเป็น phase เดียวได้หลายเกม, (ข) **1 mechanic ใหม่ = อย่างน้อย 1 phase ของตัวเอง** (เพราะเป็นส่วนที่แพงและเสี่ยงที่สุด), (ค) หมวดที่แชร์ mechanic เดียวกันจับรวม phase เดียวได้
4. **Mechanic ที่ใช้ข้ามชั้น สร้างครั้งเดียวให้ยืดหยุ่น** — 3 สายจาก `แผนเกมภาพรวม-ป1-6.md` (coding "เรียงบัตรคำสั่ง", คำนวณ "ต่อยอด ar-math", ไทม์ไลน์ "ต่อยอด ar-order") พอสร้าง engine ที่ ป.ต้นแล้ว ชั้นถัดไปแค่เพิ่มชนิดบัตร/คลังโจทย์ = phase สั้นลงมาก **ออกแบบ engine เผื่อ extension ตั้งแต่ครั้งแรกเสมอ**
5. **ดีไซน์/UI = ธีมเดียวปรับ scale อัตโนมัติ (ตัดสินใจแล้ว 2026-07-20)** — คงธีมการ์ตูนสดใสเดิมทั้ง ป.1-6 ไม่แตกโหมด/ไม่ rebrand แต่ให้ **ขนาดตัวอักษร / ความยาว-ความหนาแน่นข้อความ / จำนวน element ต่อจอ ปรับตามเนื้อหาแต่ละหมวด** — เนื้อหายาว/นามธรรมของ ป.3 ขึ้นไปได้ตัวอักษร-พื้นที่มากขึ้นอัตโนมัติ **หลักการนี้เริ่มมีผลตั้งแต่ ป.3** (ข้อความเริ่มยาว) และ**แทนที่ "phase 0 ทบทวน UI" เดิมที่เคยตั้งไว้ก่อน ป.4** — เคลียร์แล้ว ไม่มี phase 0 เป็น gate อีกต่อไป

| **ป.1** | 10 | **4 phase** | 2 (coding engine, predict-check) | ชั้นตั้งต้น — สร้าง engine ที่ใช้ต่อได้หลายชั้น |

## ป.1 — 4 phase (ชั้นตั้งต้น)

> เป้าหมายชั้นนี้: สร้าง engine ที่ใช้ต่อได้ยาว (coding, predict-check) + เก็บ quick win จากการต่อยอดเกมเดิมให้มากที่สุด

### Phase 1.1 — ต่อยอดของเดิม (quick win, ไม่มี mechanic ใหม่)
รวมทุกไอเดียที่ใช้ mechanic เดิม 100% แค่เพิ่มคลังโจทย์ — เสี่ยงต่ำ ทดสอบง่าย ควรทำก่อนเพื่อได้ momentum
- ขยาย `behavior` ให้ครบ 5 ด้าน CASEL (เพิ่มมุม "รู้จักอารมณ์ตนเอง" เลือกหน้าอีโมจิ + "เห็นใจผู้อื่น")
- ต่อยอดเกมดนตรี `skill-music1-3` (จับจังหวะ, เสียงสูง-ต่ำ/ยาว-สั้น)
- ต่อยอดเกมผสมสี `skill-mix`/`skill-mix2` (เส้น/รูปร่าง/พื้นผิว)
- ต่อยอด `pattern`/`days` (วัฏจักรธรรมชาติ, สิ่งมีชีวิต↔สิ่งแวดล้อม)
- **ขนาด: กลาง** (หลายเกมแต่ทุกเกมแค่เพิ่ม pool) — **release ได้เป็น minor version**

### Phase 1.2 — ขยาย iq + ระบบเลเวล 5 ระดับของวิชาหลัก
- ขยาย `iq1-4` ให้ครอบคลุม executive function (Simon Says ฝึก inhibitory control, สลับกฎกลางทางฝึก cognitive flexibility) — **มี mechanic ใหม่บางส่วน** (ตัวจับเวลา/สัญญาณเปลี่ยนกฎ)
- วางระบบเลเวล 5 ระดับให้ `math`/`thai`/`english` (เลเวล 1-3 = ป.1, เลเวล 4-5 = เนื้อหาเร่ง ป.2) — reuse pattern `cat.mathTiers`/`q.tier` ที่มีอยู่
- **ขนาด: กลาง-ใหญ่** — ระบบเลเวลเป็นโครงที่ชั้นอื่นใช้ต่อ ควรออกแบบให้ generic

### Phase 1.3 — 🥇 หมวดใหม่ "เรียงคำสั่งหุ่นยนต์" (coding engine — สร้างครั้งเดียวใช้ 6 ชั้น)
- mechanic ใหม่ทั้งหมด: ลากบัตรคำสั่ง (เดินหน้า/เลี้ยวซ้าย-ขวา) เรียงเป็นลำดับให้ตัวการ์ตูนไปถึงเป้าบนกริด
- **ออกแบบ engine เผื่อ extension**: ป.1 ใช้แค่ sequence แต่ต้องรองรับ loop (ป.2), เงื่อนไข (ป.3), parameter (ป.6) ในอนาคตได้โดยไม่ต้อง rewrite
- **ขนาด: ใหญ่** — เป็น mechanic ใหม่ที่ลงทุนสูงสุดของชั้นนี้ แต่คุ้มที่สุด (ใช้ต่อทั้ง 6 ชั้น) — **release เดี่ยว 1 phase**

### Phase 1.4 — หมวดใหม่วิทยาศาสตร์ (predict-check engine)
- "STEM ทดลองคิด" + "นักวิทยาศาสตร์น้อย" — 2 หมวดแชร์ mechanic เดียว (ทำนายก่อน-ตรวจทีหลัง: ลอย/จม, งอกของเมล็ด)
- engine นี้ใช้ต่อได้ที่ ป.2 (นักวิทยาศาสตร์น้อย 2) และ ป.5 (นักวิทยาศาสตร์น้อย 3)
- **ขนาด: กลาง** — mechanic ใหม่แต่ไม่ซับซ้อนเท่า coding

## ป.2 — 3 phase

> เป้าหมาย: สร้าง calculation engine (คูณ-หาร) ที่เป็นกระดูกสันหลังของคณิต ป.2-6 + extend coding/science ที่สร้างไว้แล้ว

### Phase 2.1 — 🥈 หมวดใหม่ "คูณ-หารมหัศจรรย์" (calculation engine)
- ต่อยอด mechanic `ar-math` เดิม (ลากการ์ดคำตอบ) เปลี่ยนโจทย์เป็นคูณ-หาร — **เป็นต้นสายที่ ป.3-6 ใช้ต่อ** (เศษส่วน→ทศนิยม→ร้อยละ→อัตราส่วน)
- **ขนาด: กลาง** (mechanic มีอยู่แล้ว แต่ต้องออกแบบคลังโจทย์คูณ-หารให้ครบ tier)

### Phase 2.2 — extend coding + นาฬิกา (ใช้ engine เดิม)
- ขยาย "เรียงคำสั่งหุ่นยนต์" เพิ่มบัตร "ทำซ้ำ N ครั้ง" (loop, CSTA 1A) — extend engine phase 1.3
- หมวดใหม่ "นาฬิกา+ปฏิทิน" — ต่อยอด `skill-clock` เดิม เพิ่มการอ่านปฏิทิน/ระยะเวลา
- **ขนาด: เล็ก-กลาง** (extend ล้วน)

### Phase 2.3 — ต่อยอดของเดิม + นักวิทยาศาสตร์น้อย 2
- นักวิทยาศาสตร์น้อย 2 (วัฏจักรชีวิตสัตว์ ผีเสื้อ/กบ) — extend predict-check engine phase 1.4
- ขยาย `behavior`/`iq1-4`/`skill-memory` + ดนตรี/ผสมสี (จังหวะผสม/ผสม 3 สี)
- ขยายวิชาหลักเป็น 5 เลเวล (เลเวล 4-5 = เนื้อหาเร่ง ป.3)
- **ขนาด: กลาง** (ต่อยอดล้วน)

## ป.3 — 5 phase (หมวดใหม่เริ่มหนาแน่น)

> เป้าหมาย: ป.3 เนื้อหาวิชาหลักแทบไม่ทับซ้อนกับแอปเลย ต้องเปิดหมวดใหม่มากที่สุดเท่าที่ผ่านมา — แตก phase ตาม mechanic

### Phase 3.1 — 🥈 หมวดใหม่ "เศษส่วนแสนสนุก" (แกนของ ป.3)
- extend calculation chain + เพิ่ม mechanic ย่อย "แบ่งพิซซ่า/เค้ก" (แตะ/ลากชิ้นส่วนตามเศษส่วน) — เศษส่วนเป็นแนวคิดใหม่ทั้งหมด เป็นเนื้อหาแกนของชั้น ควรทำก่อน
- **ขนาด: กลาง-ใหญ่**

### Phase 3.2 — หมวดใหม่ "ร้านค้าจำลอง" (เรื่องเงิน)
- mechanic เฉพาะตัว: ลากเหรียญ/ธนบัตรจ่ายให้พอดีราคา + ทอนเงิน
- **ขนาด: กลาง**

### Phase 3.3 — 🥉 หมวดใหม่ "เส้นเวลาประวัติศาสตร์" (timeline engine)
- mechanic ใหม่: ลากวางบนเส้นเวลา — **เป็นต้นสายที่ ป.4-6 ใช้ต่อ** (ไทม์ไลน์สุโขทัย/อยุธยา/รัตนโกสินทร์)
- **ขนาด: กลาง** (mechanic ใหม่แต่ใช้ต่อได้ยาว)

### Phase 3.4 — หมวดใหม่วิทยาศาสตร์+อังกฤษ (2 mechanic ใหม่)
- "นักสืบแม่เหล็ก" (ลากจับคู่ `ar-match` เดิม + ทำนายขั้ว N/S) + "โลกหมุน" (ลากหมุน/วางตำแหน่ง — เชื่อมกับธีมกลางวัน-กลางคืนของแอป)
- "จัดหมวดหมู่คำอังกฤษ" (ลากใส่ตะกร้าตามประเภท — mechanic ใหม่ที่ ป.4 ใช้ต่อ)
- **ขนาด: ใหญ่** (อาจแตกเป็น 3.4a/3.4b ถ้าใหญ่ไป)

### Phase 3.5 — extend coding + ต่อยอดของเดิม
- ขยาย "เรียงคำสั่งหุ่นยนต์" เพิ่มบัตรเงื่อนไข "ถ้าเจอ...ให้..." (CSTA 1B)
- ต่อยอดนาฬิกา+อังกฤษ, ดนตรี/ผสมสี, `behavior`/`iq`/`skill-memory`, วิชาหลัก 5 เลเวล
- **ขนาด: กลาง**

## ✅ เรื่องดีไซน์เด็กโต — เคลียร์แล้ว (ไม่มี Phase 0 gate อีกต่อไป)

ทุกไฟล์วิจัยตั้งแต่ ป.4 เตือนว่าเนื้อหา ป.4-6 (อายุ 9-12) ห่างจากดีไซน์ "เด็ก 5 ขวบ" เดิมมาก เดิมวางไว้เป็น "Phase 0 ทบทวน UI" ที่ต้องคุยก่อน — **ตัดสินใจแล้ว (2026-07-20): ใช้ธีมเดียวปรับ scale อัตโนมัติ** (ดูหลักการข้อ 5) จึงไม่ต้องมี phase 0 เป็น gate อีก แค่ยึดหลักการนี้ตั้งแต่ ป.3:
- **คงธีมการ์ตูนสดใสเดิมทั้งแอป** ไม่แตกโหมด/ไม่ rebrand
- **scale อัตโนมัติ**: ขนาดตัวอักษร / ความยาว-ความหนาแน่นข้อความ / จำนวน element ต่อจอ ปรับตามเนื้อหาแต่ละหมวด (หมวดข้อความยาวได้พื้นที่มากขึ้น หมวดเด็กเล็กคงเดิม)

## ป.4 — 5 phase

> เป้าหมาย: ป.4 หมวดใหม่มากที่สุด (9) + เป็นรอยต่อพัฒนาการจริง (Montessori/CASEL ขยับ) — เนื้อหานามธรรม/ข้อความยาว ใช้ scale อัตโนมัติตามหลักการข้อ 5

### Phase 4.1 — 🥈 หมวดใหม่ "ทศนิยมมหัศจรรย์" (extend calculation chain)
### Phase 4.2 — หมวดใหม่เรขาคณิต: "นักสำรวจมุม" (ลากหมุน ต่อยอด skill-clock) + "พื้นที่และเส้นรอบรูป" (grid นับตาราง — mechanic ใหม่ ป.5 ใช้ต่อ)
### Phase 4.3 — หมวดใหม่วิทยาศาสตร์: "นักสืบระบบสุริยะ" (สาย C เรียงลำดับ) + "จัดหมวดนักสืบวิทยาศาสตร์" (ลากใส่ตะกร้า extend ป.3)
### Phase 4.4 — หมวดใหม่ภาษา: "แยกจริง-แยกคิด" + "แผนภาพความคิด mind map" (mechanic ใหม่) + "ผันกริยาตามประธาน" + "ไทม์ไลน์สุโขทัย" (extend timeline)
### Phase 4.5 — "สถานการณ์ยุติธรรม" (Montessori fairness + CASEL) + extend coding (เงื่อนไขซ้อน) + ต่อยอดของเดิม + วิชาหลัก 5 เลเวล
- **หมายเหตุ**: "สถานการณ์ยุติธรรม" เป็นข้อที่ไฟล์วิจัยเองตั้งข้อสงสัยว่า mechanic ปรนัย static จะสื่อความซับซ้อนทางสังคมได้พอไหม — **ต้องพิจารณา mechanic ให้ดีตอนออกแบบ** อาจเป็นข้อที่เลื่อนออกได้ถ้ายาก

## ป.5 — 4 phase

> เป้าหมาย: ส่วนใหญ่ extend สายที่สร้างไว้แล้ว + เพิ่ม mechanic ใหม่เฉพาะ 2 จุด

### Phase 5.1 — 🥈 "เศษส่วน-ทศนิยม-ร้อยละมหัศจรรย์" (extend calculation) + "ปริมาตร-พื้นที่" (extend grid ป.4)
### Phase 5.2 — หมวดใหม่ภาษา: "จับคู่ความหมายไทยขั้นสูง" (ลากจับคู่) + "จับใจความภาษาอังกฤษ" (ปรนัยจับใจความ)
### Phase 5.3 — หมวดใหม่วิทยาศาสตร์: "โซ่อาหาร-วัฏจักรน้ำ" (mechanic ใหม่ ลากลูกศรเป็นวง) + "นักวิทยาศาสตร์น้อย 3" (extend predict-check) + "นักประวัติศาสตร์น้อย" (extend timeline)
### Phase 5.4 — extend coding (if-then หลายชั้น) + ต่อยอดของเดิม (`behavior`/`iq`/ดนตรี/ผสมสี) + วิชาหลัก 5 เลเวล

## ป.6 — 5 phase (ชั้นสุดท้าย + capstone)

> เป้าหมาย: ชั้นซับซ้อนที่สุด + ไม่มีเนื้อหาเร่งข้ามชั้น (ทุกเลเวลจากเนื้อหา ป.6 เอง) + capstone รวบยอด ป.1-6

### Phase 6.1 — 🥈 "อัตราส่วน + ห.ร.ม./ค.ร.น." (ปิดสาย calculation) + "พับกล่อง" (ลากจับคู่ รูปคลี่↔3มิติ)
### Phase 6.2 — ⭐ หมวดใหม่ "ต่อวงจรไฟฟ้า" — **หมวดที่มีแรงหนุนจากหลายมุมที่สุด** (วิชาหลัก + STEM 4.4 + Science Lab 4.6 บรรจบกัน) mechanic เฉพาะ ควรพิจารณาก่อนในชั้นนี้
### Phase 6.3 — หมวดใหม่วิทยาศาสตร์/คณิต: "ระบบย่อยอาหาร" (extend timeline/เรียงลำดับ) + "จำลองคราส" (ลากหมุน/วางตำแหน่ง)
### Phase 6.4 — หมวดใหม่ภาษา + coding: "เกมโน้มน้าวใจ" (ปรนัยจับใจความ) + "เกมบอกทิศทาง" (เชื่อมฟีเจอร์ "บ้านของหนู" ถ้ามีแล้ว) + "เรียงคำสั่งหุ่นยนต์ 3" (ปิดสาย coding ด้วย parameter, CSTA Level 2) + "ไทม์ไลน์รัตนโกสินทร์" (ปิดสาย timeline)
### Phase 6.5 — ⭐ capstone "รวมทุกอย่าง ป.1-6" — โหมดสุ่มโจทย์ทบทวนข้ามทุกชั้น (ต่อยอด `poolPick`/`tier` แต่ pool ข้ามชั้น) + ขยาย `behavior`/`iq` ระดับสูงสุด
- **หมายเหตุ**: capstone ทำได้ก็ต่อเมื่อ ป.1-5 พัฒนาจริงครบแล้ว (ต้องมีคลังโจทย์ให้สุ่ม) — เป็น phase ปิดท้ายจริงๆ

## สรุปลำดับแนะนำ (ถ้าทำเรียงจริง)

**เส้นทางหลัก (ทีละชั้น ตามที่ผู้ใช้กำหนด):**
ป.1 (4 phase) → ป.2 (3) → ป.3 (5) → ป.4 (5) → ป.5 (4) → ป.6 (5) = ~26 phase — **พัฒนา+merge สะสมทุก phase แล้ว deploy ครั้งเดียวตอนจบครบ 6 ชั้น** (ไม่ release รายเฟส)

**ทางเลือก (ถ้าอยากได้ engine ที่ใช้ซ้ำเร็วขึ้น):** ทำ phase ที่สร้าง engine หลักก่อนข้ามชั้น แล้วค่อยเติมเนื้อหา — แต่ขัดกับ "ทีละชั้น" ที่ผู้ใช้กำหนด จึงเสนอไว้เป็นทางเลือกเฉยๆ ไม่ใช่ค่าเริ่มต้น:
1. coding engine (ป.1 phase 1.3) — ใช้ 6 ชั้น
2. calculation engine (ป.2 phase 2.1) — ใช้ 5 ชั้น
3. timeline engine (ป.3 phase 3.3) — ใช้ 4 ชั้น

**ข้อเสนอเชิงกลยุทธ์**: ทำ ป.1 phase 1.1 (quick win ต่อยอดของเดิม) ก่อนสุดเสมอ เพราะเสี่ยงต่ำ ได้ผลเร็ว และพิสูจน์ว่าระบบขยายหมวด/เลเวลของแอปรองรับ scope ที่ใหญ่ขึ้นได้จริง ก่อนลงทุนกับ mechanic ใหม่ที่แพงกว่า

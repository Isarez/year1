คณิตศาสตร์ ป.1

ความมั่นใจสูงมาก — อ่านจากเอกสาร สสวท. โดยตรงทุกตัวชี้วัด
ช่องว่างหลัก: เนื้อหา ป.1 ใกล้เคียงหมวด math/ar-math เดิมมากที่สุดในบรรดา 5 วิชา แต่ยังขาดการนับทีละ 10 อย่างเป็นระบบ, มุมมองส่วนย่อย-ส่วนรวม, การวัดความยาว/น้ำหนักจริงจัง, การจำแนกรูปทรง 2 มิติ/3 มิติ และการอ่านแผนภูมิรูปภาพ — ส่วน "แบบรูป" (ค1.1) มีหมวด pattern 🔮 ครอบคลุมไว้แล้ว ไม่ต้องทำซ้ำ

2 หมวด · engine ใหม่ทั้งหมด (drag-to-bin หลายตะกร้า) · ทำหลังสุดเพราะเสี่ยงสุด
วัสดุรอบตัวจำเป็น VS อยากได้

แนะนำแยก branch feature/ ต่อเฟส (ไม่รวมเป็น branch เดียวก้อนใหญ่) เทสและขอ merge ทีละเฟสตามกติกาโปรเจกต์ — เฟส 1 จบก่อนค่อยเริ่มเฟส 2

ประเด็นสถาปัตยกรรมที่ต้องตัดสินใจก่อนเริ่ม

นี่คือจุดใหญ่สุด: แอปกำลังขยายจาก "เตรียมสอบเข้า ป.1 สำหรับเด็ก 5 ขวบ" ไปเป็น "รองรับหลักสูตร ป.1-6" — ต้องตัดสินใจว่าจะจัดวางเนื้อหาใหม่บนหน้าเลือกหมวดยังไง ก่อนเริ่มเขียนโค้ดจริง

ทางเลือกที่เป็นไปได้
A

เพิ่ม grade selector หน้าแรก คล้าย .grade-chip ที่โหมดครูมีอยู่แล้ว กรอง #cat-grid* ตามชั้นที่เลือก เก็บชั้นที่เลือกไว้ต่อโปรไฟล์เด็ก (ขยาย object โปรไฟล์เดิมใน localStorage)

B

แยก "เตรียม ป.1" ออกจาก "ป.1 หลักสูตรทางการ" เป็นคนละชุดที่เลือกเล่นได้อิสระ ไม่ผสมกัน — ของเดิมทั้งหมด (30 หมวด) ยังอยู่เหมือนเดิมเป๊ะ ไม่กระทบ progress/สติกเกอร์ผู้ใช้เก่า

C

รวมเป็นชุดเดียวต่อชั้น ยุบของเดิมเข้ารวมกับเนื้อหาใหม่ตามหลักสูตรทางการทั้งหมด — เปลี่ยนโครงสร้างมากสุด เสี่ยงกระทบผู้ใช้เดิมมากสุด

ความเห็นเบื้องต้น: A+B ร่วมกันดูปลอดภัยสุด — มี grade selector แบบ A แต่ให้ "เตรียมสอบ ป.1" เป็นตัวเลือกแยกอยู่เสมอ (ไม่ใช่แค่ history) กันกระทบผู้ใช้เดิมตามแบบ B ยังไม่ฟันธง รอฟังความเห็นก่อนเริ่มออกแบบ UI จริง

ข้อเสนอเพิ่ม: ใช้ prefix p1-, p2- … นำหน้า id หมวดทุกอันที่มาจากหลักสูตรทางการ (ต่างจาก id เดิมที่ไม่มี prefix) เพื่อกันชนกับหมวดเดิมและแยกกรองตามชั้นได้ง่ายเมื่อทำ ป.2-6 ต่อ — อีโมจิทั้ง 11 ตัวที่เสนอ (🔷📏🧚🙋🏞️🫀🧱🌠👪🛒🧭) ตรวจแล้วว่าไม่ซ้ำกับอีโมจิที่ใช้อยู่ในแอปปัจจุบันทั้งหมด

สรุปก่อนเริ่มลงมือ

เอกสารนี้ครอบคลุมเฉพาะ ป.1 ตามคำแนะนำในไฟล์วิจัย mechanic ที่ให้เริ่มจากชั้นที่เนื้อหาใกล้เคียงแอปปัจจุบันที่สุดก่อน ป.2-6 จะทำเป็นรอบถัดไปหลัง ป.1 ผ่านการทดสอบจริงแล้ว ยังไม่มีการแก้โค้ดใดๆ ในรอบนี้ทั้งสิ้น

  1. ยืนยันขอบเขต — จะเริ่มทั้ง 11 หมวดพร้อมกัน หรือเริ่มแค่เฟส 1 (7 หมวด reuse ล้วน) ก่อน
  2. ตอบประเด็นสถาปัตยกรรม grade selector (ตัวเลือก A/B/C ด้านบน) อย่างน้อยคร่าวๆ ก่อนเริ่ม UI หน้าแรก
  3. ยืนยัน naming convention p1- prefix
  4. เมื่อ cf แล้วจะสร้าง branch feature/p1-curriculum (หรือแยกตามเฟส) ตามกติกาโปรเจกต์ทันที ไม่ commit ตรง main
⏳ รอ "cf" เพื่อเริ่มลงมือ

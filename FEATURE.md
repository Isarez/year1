# ฟีเจอร์ที่วางแผนไว้

ไฟล์นี้เก็บสเปคฟีเจอร์ที่ตกลงจะทำแล้ว (ต่างจาก [IDEA.md](IDEA.md) ที่เป็นไอเดียเกม/หมวดใหม่ที่ยังไม่แน่นอน) — รายละเอียดในนี้ควรชัดเจนพอเอาไปพัฒนาได้ทันที

**กติกา** (ดู [CLAUDE.md](CLAUDE.md) ด้วย):
- แต่ละฟีเจอร์เก็บเป็น 2 ส่วนเสมอ: **ชื่อฟีเจอร์** (สั้นๆ ใช้เป็น `##` heading) และ **รายละเอียด** (อธิบายกลไก/วิธีทำงานให้ชัดพอเอาไปทำต่อได้)
- เมื่อฟีเจอร์ไหนถูกพัฒนาเสร็จและ merge เข้าแอปจริงแล้ว ให้ลบออกจากไฟล์นี้ทันที
- ก่อนเริ่มพัฒนาฟีเจอร์ไหน ให้ถามผู้ใช้ก่อนเสมอว่า "จะทำฟีเจอร์นี้เลยไหม?" รอผู้ใช้ตอบ **"cf"** ก่อน แล้วอ่านรายละเอียดอีกครั้งอย่างละเอียด ถ้ามีจุดไหนไม่ชัดเจน/สงสัย/ขาดข้อมูลที่ต้องตัดสินใจ (เช่น "หมายเหตุ/ข้อจำกัดทางเทคนิค" ที่ระบุไว้ในฟีเจอร์นั้น) ให้ **ถามผู้ใช้ก่อนลงมือสร้าง** อย่าเดาเอง — ถ้าข้อมูลครบไม่มีข้อสงสัยแล้วให้เริ่มทำได้เลยโดยไม่ต้องถามซ้ำ

## ย้ายข้อมูล
รายละเอียด:

**Export (อยู่ในโมดัล "จัดการข้อมูล" เดิม — `#clear-modal`, เปิดจากไอคอน 🗑️ บน header):**
- เพิ่มปุ่ม "ย้ายข้อมูล" ในโมดัลนี้
- กดแล้ว export เฉพาะข้อมูลของ **เด็กที่กำลังเลือกอยู่ตอนนั้น** (`activeChild`) เท่านั้น — ดึงจาก localStorage ทั้งข้อมูลโปรไฟล์เด็ก (`children` array เฉพาะ record ของเด็กคนนี้) และ progress ของเด็กคนนั้น (`p1quiz_progress_<childId>`)
- Encode เป็น JSON, UTF-8
- ใส่ signature บางรูปแบบต่อท้าย/แนบไปกับข้อมูลด้วย เพื่อใช้ verify ตอน import (เช่น checksum/HMAC ของเนื้อหาไฟล์)
- ตั้งชื่อไฟล์เป็น `owlkids_data_{{uuid}}.json` (สุ่ม uuid ใหม่ทุกครั้งที่ export)
- ดาวน์โหลดไฟล์ลงเครื่องผู้ใช้ (ผ่าน `Blob` + `<a download>` ตามมาตรฐานเว็บ)

**Import (อยู่หน้าแรก — หน้าเลือกโปรไฟล์เด็ก `#child-select-view`):**
- เพิ่มปุ่ม "นำเข้าข้อมูลเด็ก" อยู่ใกล้ปุ่ม "+ เพิ่มเด็กคนใหม่" แต่ให้ดูเด่นน้อยกว่าปุ่มนั้น (เช่น ตัวเล็กกว่า/สีอ่อนกว่า/ไม่ใช่ปุ่มหลัก)
- กดแล้วเปิด file picker ให้เลือกไฟล์จากเครื่องผู้ใช้
- อ่านไฟล์ → **verify signature ก่อนเสมอ** — ถ้า signature ไม่ถูกต้อง/ไฟล์เสียหาย ให้ขึ้นแจ้งเตือนผู้ใช้ (ไฟล์ไม่ถูกต้อง ไม่สามารถนำเข้าได้) แล้วหยุด ไม่ import
- ถ้า signature ผ่าน → decode UTF-8 → parse เป็น JSON → นำเข้าข้อมูลเด็ก (และ progress ของเด็กคนนั้น) ลง localStorage ของเครื่อง
- **render หน้าเลือกโปรไฟล์ใหม่ทันที** ให้เห็นเด็กที่เพิ่ง import เข้ามาปรากฏในรายการ
- เมื่อ import สำเร็จ ให้ถามผู้ใช้ว่า "ต้องการลบไฟล์ข้อมูลเด็กที่เพิ่ง import ออกจากเครื่องไหม?" — ถ้าผู้ใช้ confirm ให้ลบไฟล์นั้นออกจากเครื่องผู้ใช้

**หมายเหตุ/ข้อจำกัดทางเทคนิคที่ต้องคุยกับผู้ใช้ก่อนเริ่มทำ:**
- การลบไฟล์ที่เพิ่ง import ออกจากเครื่องผู้ใช้ **ทำไม่ได้ในทุกเบราว์เซอร์** ด้วยเหตุผลด้านความปลอดภัย — เบราว์เซอร์ทั่วไปที่เลือกไฟล์ผ่าน `<input type="file">` แบบมาตรฐานจะไม่ให้ JavaScript เขียน/ลบไฟล์บนดิสก์ของผู้ใช้ได้เลย มีแค่ File System Access API (`showOpenFilePicker` + file handle ที่ขอสิทธิ์เขียนได้) ที่ทำได้ แต่รองรับเฉพาะ Chrome/Edge บนเดสก์ท็อปเท่านั้น (Safari/Firefox/มือถือไม่รองรับ) — ต้องตกลงกับผู้ใช้ก่อนว่าจะ (ก) ใช้ File System Access API เฉพาะเบราว์เซอร์ที่รองรับ แล้ว fallback เป็นแค่แจ้งผู้ใช้ให้ลบเองในเบราว์เซอร์อื่น หรือ (ข) ตัดฟีเจอร์ลบไฟล์อัตโนมัติออก เปลี่ยนเป็นแค่ข้อความแนะนำให้ผู้ใช้ลบไฟล์เองจากโฟลเดอร์ดาวน์โหลด
- ต้องเลือกรูปแบบ signature ที่ใช้ (เช่น checksum ธรรมดา (ตรวจจับไฟล์เสียหาย/ไม่ใช่ไฟล์จากแอปนี้) หรือ signature ที่ป้องกันการปลอมแปลงเนื้อหาได้จริงด้วย — ถ้าต้องการอย่างหลังต้องคุยเรื่อง key/secret ที่ใช้เซ็นด้วย เพราะเป็น static site ไม่มี backend เก็บ secret ให้ปลอดภัยจริงๆ ได้)
